## 项目经历

### PRJ_01 - 晶圆制造缺陷分析系统

| 属性 | 详情 |
|------|------|
| **时间** | 2024.09 - 2025.07 |
| **角色** | AI产品经理 / 全栈开发 |
| **公司/团队** | 浙江大学 · 三脚架队（3人团队）|
| **地点** | 杭州 |
| **状态** | 已上线（校企合作） |
| **标签** | Python, ResNet50, CLIP, YOLOv8, SAM, React, Flask |

#### 项目概述
针对产线良率分析依赖人工、缺乏根因追溯的痛点，主导从 0 到 1 设计"检测-诊断-溯源"闭环系统。与校企合作公司合作，使用真实产线数据训练，将模型能力封装供员工调用分析。

#### 核心贡献

**产品设计**
- 主导从 0 到 1 设计"检测-诊断-溯源"闭环系统，撰写完整 PRD 文档
- 定义四大核心模块：缺陷检测、缺陷分割、缺陷分类、根因分析
- 完成交互原型开发与全栈集成

**全栈开发**
- 前端：React + Tailwind CSS，设计并实现高保真 UI 界面
- 后端：Flask 搭建 API 服务，连接算法模型与前端交互

#### 技术架构

**四大核心模块**
| 模块 | 技术 | 功能 |
|------|------|------|
| 缺陷检测 | YOLOv8 | 快速定位晶圆图中的缺陷位置 |
| 缺陷分割 | SAM | 精确分割缺陷边界，计算面积占比 |
| 缺陷分类 | ResNet50 | 12类晶圆缺陷分类，Top-1准确率96.2% |
| 根因分析 | CLIP+OCR | 视觉问答+知识库匹配，输出缺陷成因 |

**技术难点与解决方案**
- **晶圆图与自然图像分布差异大**：晶圆图具有复杂的规律性背景图案，缺陷尺度差异大，传统视觉模型效果差
- **工业少样本问题**：仅2000+张标注数据，通过迁移学习+数据增强解决
- **实时性要求**：推理速度优化至0.03秒，满足产线实时检测需求

#### 技术亮点
- **多模态检索**：设计 CLIP 视觉问答策略，实现 Top-1 准确率 96.2%及 0.03 秒实时推理
- **闭环系统**：从缺陷发现到根因追踪的全流程自动化
- **校企合作落地**：使用真实产线数据训练，模型能力已封装供员工调用

#### 项目成果
| 指标 | 数值 | 说明 |
|------|------|------|
| 人工标注成本降低 | 90% | 显著提升产线效率 |
| Top-1 准确率 | 96.2% | 12类缺陷分类准确率 |
| 实时推理 | 0.03s | 满足工业实时性要求 |
| 数据规模 | 2000+张 | 校企合作真实产线数据 |
| 荣誉奖项 | 1项 | 华为杯全国一等奖 |

#### 比赛经历
- **华为杯第七届中国研究生人工智能创新大赛**
- 决赛采取线下答辩形式，评委重点关注项目落地情况
- 问题集中在：是否已在产线上部署、如何与企业现有系统集成
- 回答：使用校企合作真实数据训练，模型能力已封装供员工调用分析

#### UI展示
> 界面截图可在详情页展示，包括系统架构图、四大核心功能界面等

#### 复盘总结
- ✅ **商业价值落地**：与校企合作，使用真实产线数据，将人工标注成本降低 90%
- ✅ **技术与业务平衡**：充分考虑精度与产线效率的平衡，确保系统实用性
- ✅ **产品思维实践**：从需求调研到产品交付的完整流程，积累了AI产品化经验

## 实习经历

### EXP_01 - AI产品经理实习生

| 属性 | 详情 |
|------|------|
| **时间** | 2025.12 - 至今 |
| **角色** | AI产品经理实习生 |
| **公司/团队** | 北京智谱华章科技股份有限公司 · 泛互联网业务部 |
| **地点** | 杭州 |
| **状态** | 进行中 |
| **标签** | 大模型产品, RAG知识库, 智能体交互, 数据可视化 |

#### 工作概述
参与杭州城投集团"城智大模型体系"一期建设，负责安居集团知识库构建、权限配置，并主导城投展示大屏的设计与智能体交互搭建。

#### 核心职责

**1. 安居集团知识库构建**
- 对比多种文档切片策略（固定长度切片、语义切片、递归字符切片等），最终选择**语义切分+固定窗口混合策略**，在保证语义完整性的同时控制检索粒度
- 设计知识库分层结构，将安居集团的政策文件、业务文档、FAQ等按主题域分类，提升检索准确率
- 负责知识库权限配置，实现不同角色（管理员、业务员、访客）的差异化访问控制
- 搭建知识质检流程，通过小批量测试验证检索效果，持续优化切片参数

**2. 城投展示大屏主导设计**
- 使用 **Axure** 设计交互原型，使用 **Figma** 设计高保真 UI 图
- 负责大屏每个页面的整体设计与布局，确定核心呈现指标与交互模块
- 撰写每个页面的讲解文字稿，确保展示内容的准确性与叙事连贯性
- 设计大屏内嵌智能体的交互响应逻辑，包括用户意图识别、答案生成、结果呈现等环节

**3. 智能体交互搭建**
- 采用 **Vibe Coding** 模式快速构建高保真可交互 HTML 原型，验证交互逻辑可行性
- 与前端开发团队对接，将原型转化为可落地方案
- 使用智谱内部智能体平台搭建完整工作流，并封装成 API，支持 curl 等形式调用

#### 工作亮点
- **智能体交互设计**：从0到1设计大屏智能体交互流程，实现了用户自然提问与城投数据的无缝对接
- **快速原型验证**：采用 Vibe Coding 模式快速构建高保真原型，有效降低了与开发团队的沟通成本
- **知识库策略优化**：通过对比多种切片策略并选择混合方案，在检索召回率与响应速度间取得平衡

#### 工作成果
| 指标 | 数值 | 说明 |
|------|------|------|
| 知识库文档 | 待补充 | 安居集团政策与业务文档 |
| 大屏页面 | 17个 | 1首页+6弹窗+4子场景+6子页面 |
| 智能体工作流 | 1套 | 完整的问答交互流程+API封装 |
| 设计工具 | Axure+Figma | 原型与UI设计 |

#### 复盘总结
- ✅ **To B 项目落地经验**：参与了大型国企的AI项目落地，理解了传统企业数字化转型的实际需求
- ✅ **产品全流程参与**：从需求调研、方案设计到落地交付，积累了完整的产品工作经验
